Plugin de Autenticação

Poderá escolher como é que os utilizadores desta instituição vão efectuar a sua autenticação no site. As instituições devem ter pelo menos um método de autenticação definido.

O plugin de autenticação interna faz a verificação dos nomes de utilizador/palavra-chave na própria base de dados do site. Os utilizadores têm que ter uma conta para que funcione o qual pode ser feito permitindo resgistos através de uma formulário de inscrição, ou usando o Adicionar Utilizadores or Adicionar Utilizadores por CSV ecrãs.

O método de autenticação nula permite que qualquer utilizador tenha acesso. Os utilizadores podem escolher qualquer nome de utilizador e palavra-chave. Isto apenas poderá ser utilizado para testes!

Os outros métodos de autenticação, geralmente autenticam-se por uma fonte externa, como um servidor LDAP ou aplicativo remoto. Estes têm as suas próprias definições, que poderá configurar numa janela popup quando adicionadas.

Uma configuração que eles têm em comum é que contas podem ser criadas automaticamente no Mahara quando um utilizador acede pela primeira vez (ao contrário das contas criadas anteriormente pelo administrador, o qual só depois é que poderiam efectuar o seu acesso).
